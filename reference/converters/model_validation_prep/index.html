    ModelValidationPrep 

    ModelValidationPrep(
    pydantic_model: DynamicRecordTypeModel,
    logger: Logger | None = None,
    require_date_object: bool = False,
)

    Class for preparing a Heurist record's details for validation in the record type's Pydantic model.

    Parameters:

    • pydantic_model

      (DynamicRecordTypeModel)

      Pydantic model.

    • logger

      (Logger, default: None )

      Logger for record model validation.

    • require_date_object

      (bool, default: False )

      Whether to require date data fields to have a dictionary / JSON object, rather than a simple value. Defaults to False.

    Methods:

    • aggregate_details

      Resolve the problem of Heurist exporting a detail type's repeated values as separate JSON objects (dictionaries) by aggregating all details by the detail type.

    • convert_date_to_pydantic_kwarg

      Convert an aggregation of date details (by detail type) into a key-value pair for Pydantic field validation. Try to create key-value pairs for a generic field annotation (array of the earliest and latest datetime objects) and a temporal annotation (raw Heurist JSON object).

    • convert_enum_to_pydantic_kwarg

      Convert an aggregation of vocabulary terms (by detail type) into a key-value pair for Pydantic field validation. Try to create key-value pairs for a generic field annotation (the vocabulary term label) and a supplemental annotation (foreign key for vocabulary term, reference "trm" table).

    • convert_generic_to_pydantic_kwarg

      Convert an aggregation of vocabulary terms (by detail type) into a key-value pair for Pydantic field validation.

    • get_generic_key

      Generate a Pydantic field validation alias for any data type.

    • get_temporal_key

      Generate a Pydantic field validation alias for a date detail's supplemental column (raw JSON object of Heurist date object).

    • get_term_key

      Generate a Pydantic field validation alias for a vocabulary detail's supplemental column (foriegn key reference to vocabulary term in "trm" table).

    • is_plural

      By the detail type ID, determine if the detail allows for repeated values.

    • log_error

      Including the detail's type ID and the exception's error message, log an error message if the RecordModeler was instantiated with a logger.

    • log_message

      Using the problematic detail's type ID and the exception's error message, construct a log message.

    • log_warning

      Including the detail's type ID and the exception's error message, log a warning message if the RecordModeler was instantiated with a logger.

    • valid_heurist_date_objects

      Iterating through date detail values, log a warning if any value is not a Heurist date object.

    convert_date_to_pydantic_kwarg 

    convert_date_to_pydantic_kwarg(
    dty_id: int, details: list[dict]
) -> dict

    Convert an aggregation of date details (by detail type) into a key-value pair for Pydantic field validation. Try to create key-value pairs for a generic field annotation (array of the earliest and latest datetime objects) and a temporal annotation (raw Heurist JSON object).

    If strictly validating the date fields, requiring that all values be             a Heurist JSON object (Python dictionary), and if the validation fails,            only prepare a key-value pair for the generic field annotation.

    Parameters:

    • dty_id

      (int)

      ID of the Heurist detail.

    • details

      (list[dict])

      List of aggregated details by detail type.

    Returns:

    • dict ( dict )

      Pydantic key-value pair for field validation.
